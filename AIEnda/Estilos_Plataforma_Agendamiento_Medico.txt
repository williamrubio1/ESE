Guía de Estilos - Plataforma de Agendamiento Médico
===============================================

1. 🎨 Principios de Diseño Visual
---------------------------------
- **Claridad**: Cada pantalla debe tener un propósito claro. Evitar la sobrecarga de información.
- **Minimalismo**: Mostrar solo lo esencial. Usar espacios en blanco para separar secciones.
- **Profesionalismo**: Diseño sobrio, limpio y moderno. Nada de elementos recargados o infantiles.

2. 🌈 Paleta de Colores y Tipografía
-------------------------------------
- **Colores**:
  - Primario: Azul médico (#007BFF)
  - Secundario: Gris oscuro (#343A40)
  - Fondo: Blanco (#FFFFFF)
  - Énfasis: Verde éxito (#28A745), Rojo alerta (#DC3545), Amarillo advertencia (#FFC107)
- **Tipografía**:
  - Fuente principal: 'Inter', 'Roboto' o 'Open Sans'
  - Tamaños:
    - Títulos: 20-24px
    - Subtítulos: 16-18px
    - Texto general: 14-16px
  - Contraste alto para accesibilidad (texto oscuro sobre fondo claro)
  - Cumplimiento WCAG 2.1 nivel AA: ratio mínimo de contraste 4.5:1 para texto normal, 3:1 para texto grande.
  - Soporte de navegación por teclado (focus visible) y atributos `aria-label` en todos los controles interactivos.

3. ⚛️ Componentes Dinámicos en React
-------------------------------------
- **Calendarios interactivos**:
  - Vista semanal y mensual.
  - Citas codificadas por color según estado (disponible, agendada, cancelada, atendida).
  - Posibilidad de hacer clic en un bloque para agendar o ver detalles.

- **Dashboards**:
  - Tarjetas con métricas clave (citas del día, cancelaciones, ocupación).
  - Gráficas de barras o circulares para visualización rápida.

- **Botones y acciones**:
  - Botones grandes, con íconos y colores claros.
  - Acciones principales destacadas (ej. "Agendar cita", "Confirmar llegada").

4. 🧭 Usabilidad
----------------
- **Flujo jerárquico**:
  - Primero se elige la sede → especialidad → médico → fecha → hora → **confirmación/resumen**.
  - El último paso del wizard muestra un resumen completo de la cita antes de confirmar.
    El usuario debe revisar y aceptar explícitamente antes de que la cita quede registrada.
- **Navegación intuitiva**:
  - Menú lateral fijo para roles administrativos.
  - Navegación tipo wizard para pacientes (paso a paso), con indicador de progreso visible.
- **Vistas**:
  - Vista semanal para médicos.
  - Vista diaria para recepción.
  - Vista de historial para pacientes.
  - Vista de reagendamiento para recepción: pantalla única organizada en columnas o secciones
    jerárquicas (izquierda: búsqueda y cita actual; derecha: médicos disponibles y horarios).
    Todo el flujo ocurre sin cambiar de pantalla. Confirmación con botón destacado al pie.

5. 💡 Experiencia de Usuario
----------------------------
- **Interacciones rápidas**:
  - Carga dinámica sin recargar la página (React SPA).
- **Feedback visual inmediato**:
  - Skeleton loading (esqueletos de contenido) para dashboards, calendarios y listas de citas.
    Preferir skeleton sobre spinners genéricos para dar percepción de carga más rápida.
  - Animaciones suaves al agendar o cancelar.
- **Notificaciones claras**:
  - Toasts o alertas flotantes para confirmar acciones (éxito, error, advertencia).

6. 📱 Responsive y Modo Oscuro
-----------------------------
- **Mobile-first**: el diseño se construye primero para móvil y luego se adapta a tablet y escritorio.
  Los pacientes usan mayoritariamente celular; todas las vistas del paciente deben ser
  completamente operables en pantallas desde 360px.
- **Modo oscuro**: toggle manual en la interfaz + respeto automático de la preferencia del sistema
  (`prefers-color-scheme: dark`). Definir variantes de color para el modo oscuro en el mismo
  sistema de diseño (CSS variables / tokens).

7. 🚨 Estados Vacíos y Panel de Alertas
--------------------------------------
- **Estados vacíos**: toda pantalla que muestre listas o calendarios debe tener un estado vacío
  diseñado explícitamente (ej. "No tienes citas agendadas", "Sin médicos disponibles hoy").
  Usar ilus tración simple o icóno + texto descriptivo + acción sugerida cuando aplique.
- **Panel de alertas para administrador (espacios vacíos ≮ 6 h)**:
  - Sección destacada en el dashboard del administrador con alerta visual (color ámbar/advertencia).
  - Muestra tarjetas por cada turno libre detectado: médico, especialidad, sede, fecha y hora.
  - Acciones disponibles directamente desde la tarjeta: reasignar cita existente o cerrar alerta.
  - Diseñado para incorporar en el futuro el botón "Notificar usuarios" que disparará
    el flujo multicanal (WhatsApp, n8n, plataforma web) cuando ese módulo esté disponible.

8. 🏥 Estilo General
---------------------
- Apariencia de sistema profesional de salud.
- Inspiración en plataformas como Doctolib, Doctoralia, o agendamientos hospitalarios.
- Consistencia visual en todos los módulos.
- Adaptado en orden de prioridad: móvil → tablet → escritorio.
